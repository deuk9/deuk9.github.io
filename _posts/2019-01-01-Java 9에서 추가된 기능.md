# 1. try-with -resources
- try catch 문 밖에서 선언한 변수를 closable 구현 가능
- final이거나 사실상 final(변수에 새로운값을 할당하지 않음)에 사용 가능함.

```java
public void getResource() throws IOException {  
  
    FileInputStream fileInputStream = new FileInputStream("name");  
    final FileInputStream fileInputStream2 = new FileInputStream("test");  
  
    try (fileInputStream;  
         fileInputStream2) {  
  
    }  
  
}
```

# 2. interface 

```java
public interface Test {  
  
  
    default void print(){  
  
    }  
    private void abc() {  
        System.out.println("test");  
    }  
}
```
- 위와 같은 인터페이스에서 함수를 추가하면 구현체들은 모두 호환성이 깨짐
- `default` 함수를 통하여 어느정도 해결할 수 있다. 
- default 함수가 여러개 생기다보니 동일한 로직들이 생기기 시작함
- 동일한 로직들을 private 함수를 이용하여 사용할 수 있다. 

# 3.  Collection 기능 추가 

```java
//List 
List<Integer> oldList = Arrays.asList(1,2);
List<Integer> newList = List.of(1,2);

//Set
Set<Integer> oldSet = new HashSet<>(Arrays.asList(1,2));
Set<Integer> newSet = Set.of(1,2);

Map<String, Integer> oldMap = new HashMap<>();
oldMap.put("A", 1);
oldMap.put("B", 2);

Map<String,Integer> newMap = Map.of("A" , 1, "B");
Map.ofEntries(entry("A", 1), entry("B", 2));
```

- **of** 로 만들어진 collection 은 불변임
- 몇몇 구현체는 메모리 효율적인  구조를 갖는다.


# 4. Optional

## 4.1 ifPresentOrElse
```java
public void optionalSample(Optional<String> str) {  
  
    str.ifPresentOrElse(  
            System.out::println,  
            () -> {  
                System.out.println("널주의~");  
            });  
  
}
```
- 값의 유무에 따른 로직구현

## 4.2 or
```java
public void optionalSample(Optional<String> str) {  
	str.or(() -> Optional.of("test"));
}
```
- 값이 없으면 다른 optional 을 반환한다.


# 5. Stream
## 5.1 takeWhile

```java
Stream.of(10, 5, 15, 20)  
        .takeWhile( num -> num <= 10)  
        .collect(Collectors.toList());
```
- 조건에 만족할때까지 데이터를 취하다가 조건이 false인 값 이후로는 모든값을 버린다.
- 데이터 순서에 따라 결과값 달라지니 주의해야함.
## 5.2 dropWhile

```java
Stream.of(10, 5, 15, 20)  
        .dropWhile( num -> num >= 10)  
        .collect(Collectors.toList());
```
- 조건이 true일때 값을 계속버리다가 false 이후의 값을 모두 남긴다.


# 6. CompletableFuture

## 6.1 timeout

```java
CompletableFuture.runAsync(() -> {  
    try {  
        Thread.sleep(10000);  
    } catch (InterruptedException e) {  
  
    }  
}).orTimeout(1, TimeUnit.SECONDS);
```
- timeout 조건 추가

## 6.2 delayedExecutor

```java
public void delayedExecutor() {  
    Executor executor = CompletableFuture.delayedExecutor(1, TimeUnit.SECONDS);  
  
    CompletableFuture.runAsync(() -> {  
        try {  
            Thread.sleep(10000);  
        } catch (InterruptedException e) {  
        }  
    }, executor);  
}
```
- 지연 처리

# 7. 프로세스 API

```java
public void processHandle(){  
    ProcessHandle.current().pid();  
}
```
- native 프로세스를 제어할 수 있는 인터페이스
- 위의 샘플은 현재 프로세스의 pid를 가져올 수 있음.
